Allgemeines

WebDAV (Web-based Distributed Authoring and Versioning) ist ein offener Standard zur Bereitstellung von Dateisystemen per HTTP. Als Modul lassen sich mit dieser Erweiterung bei Seanox Devwex virtuelle Verzeichnisse für verschiedene Zielsystem einrichten. Bereits enthalten sind Stores zur als Abstraktion für lokale Dateisysteme und Zip-Archive. Alle unterstützen einen Vollzugriff mit Lesen, Schreiben, Erstellen, Ändern, Verschieben und Löschen und bieten individuelle Konfiguration und Sharing. Mit der enthaltenen API lassen sich zudem mit Java weitere Stores für neue Zielsystem, z.B. Datenbanken, FTP und SCP, implementieren und einbinden..

Merkmale

Kompakt

Das aus nur einer Datei bestehende Modul lässt sich schnell und problemlos einrichten.

HTTP

Das Modul unterstützt WebDAV Level 2 und damit die dafür erforderlichen Methoden OPTIONS, PROPFIND, HEAD, GET, PROPPATCH, MKCOL, PUT, COPY, MOVE und DELETE.

Virtualisierung

Im Lieferumfang enthalten sind bereits zwei Stores, welche die Virtualisierung vom lokalen Dateisystem und von Zip-Archiven unterstützen. In allen Stores wird ein Vollzugriff mit Lesen, Schreiben, Erstellen, Ändern, Verschieben und Löschen unterstützt.

Sicherheit

Das Modul lässt sich für eine gesicherte Datenübertragung in Verbindung mit SSL/TLS und Basic- / Digest- Access Authentication verwenden. Für Stores kann bewusst das Schreibrecht gesetzt bzw. entzogen werden.

Erweiterbar

Weitere Stores für andere Zielsysteme, z.B. Datenbanken, FTP oder SCP, lassen sich mit Java über die im WebDAV-Modul enthaltenen API erstellen.

Installation

Mit dem Modulkonzept von Devwex kann nach dem Entpacken die enthaltene Verzeichnisstruktur komplett in das Programmverzeichnis ./devwex des Servers verschoben werden.

Für die manuelle Installation, wird die nach dem Entpacken enthaltene Dateien seanox-webdav.jar an einer beliebigen Stelle im Dateisystem abgelegt. Vorzugsweise sollte für Module das Bibliotheksverzeichnis ./devwex/libraries verwendet werden. Die Anbindung an den Server erfolgt als Modul, welches sich im Klassenpfad (Classpath) des Servers befinden muss.

Devwex verfügt über eine automatische Erweiterung des Klassenpfads, für die mit dem Serverstart angegebenen Bibliotheksverzeichnisse. Somit werden dort abgelegte Module automatisch übernommen.

Die Konfiguration des Moduls erfolgt über die Konfigurationsdatei devwex.ini.

Die Ausführung und Anbindung von WebDAV kann mit einer Vielzahl von Parametern konfiguriert werden. Diese lassen sich direkt mit der Moduldeklaration angegeben werden oder über die Verwendung des Parameters EXTENDS in einen separaten Sektion auslagern.

Konfiguration des Moduls allgemein

[SERVER:X:BAS]                           
  METHODS = OPTIONS HEAD GET POST GATEWAY

Konfiguration vom FileStore

[SERVER:X:REF]                                                  
  EXAMPLE-F = /example-file/ > com.seanox.module.http.Context
            + [context:webdav-1]                                
            + [class:com.seanox.webdav.Connector]               
            + [storage:com.seanox.webdav.storage.FileStore]     
            + [root:/document] [readonly:off] [M]

Probleme

Virtuelle Pfade von Modulen, welche als Referenz (SERVER/VIRTUAL:XXX:REF) eingerichtet werden, werden vom Server absolut betrachtet. Daher sollte hier besonders auf ein endendes Slash bei der Definition virtueller Pfade geachtet werden. Im Beispiel von /d als virtueller Pfad, würde der Server bei der Anfrage nach /documents als Wurzel /d und als Pfad ocuments interpretieren. Da die Wurzel nicht mit Slash endet und der Pfad nicht Slash beginnt würde diese Anfrage als ungültiger Pfad betrachtet und mit Status NOT_FOUND (404) beantwortet werden.

Das Modul unterstützt keine Versionierung, die bei WebDAV möglich ist.

Das bewusste Sperren (LOCK) und Entsperren (UNLOCK) von Ressourcen wird nicht unterstützt.

Alle Zugriffe werden als Transaktion betrachtet, die jedoch in der Ausführungszeit (Timeout) nicht begrenzt werden.

Der Zugriff auf Ressourcen mit "+" im Namen muss URL-komform mit "%2B" erfolgen. Einige WebDAV-Clients haben hier Probleme.

Methoden

Das WebDAV-Modul stellt die gängigsten Methoden, die nachfolgend erläutert werden, zur Verfügung.
Methode
Beschreibung
OPTIONS
Rückgabe der für die angegebene Ressource zulässigen WebDAV-Methoden.
PROPFIND
Rückgabe der per XML angefordert Eigenschaften einer Ressource bzw. eine detailliertere XML Beschreibung bei Collections (Verzeichnissen).
HEAD
Rückgabe der Metadaten zu einer Ressource als HTTP-Header.
GET
Rückgabe vom Inhalt einer angeforderten Ressource. Handelt es sich dabei um ein Verzeichnis, wird dieses als navigierbares HTML-Dokument mit der Verzeichnisstruktur zurückgegeben.
PROPPATCH
Setzt die per XML übergeben Eigenschaften für einzelne oder einer Menge von Ressourcen.
MKCOL
Erstellt eine Collection (Verzeichnis). Die Übermittlung weiterer Eigenschaften und Strukturinformationen werden nicht unterstützt.
PUT
Erstellt eine Ressource (Datei) oder überschreibt den Inhalt einer bestehenden.
COPY
Kopiert eine Ressource (Datei) oder eine komplette Collection (Verzeichnis).
MOVE
Verschiebt eine Ressource (Datei) oder eine komplette Collection (Verzeichnis).
DELETE
Löscht eine Ressource (Datei) oder eine komplette Collection (Verzeichnis).

Entwicklung

Für die Entwicklung von Java Anwendungen wird bei Seanox Eclipse () eingesetzt. Nachfolgend wird die Einrichtung des Projekts für diese IDE beschreiben. Bitte haben Sie Verständnis dafür, das Seanox nur eine Entwicklungsumgebung favorisiert.
Wenn Sie das Projekt unter einer anderen verwenden, können Sie durch das Zusenden einer Anleitung bzw. Konfiguration an developing@seanox.de beim Ausbau des Manuals helfen.
